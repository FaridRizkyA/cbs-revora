GENERAL RULES
==================================================

1. Use FIXED UUID values (NOT gen_random_uuid()) so relationships stay consistent across seed files.

2. Follow existing naming conventions:
- tbl_roles
- tbl_users
- tbl_members
- tbl_products
- tbl_product_batches
- tbl_inventory_units
- tbl_external_incomes
- tbl_member_savings

3. Existing schema conventions:
- primary keys: id_xxx
- foreign keys: id_xxx
- status flag: is_active CHAR(1) with Y/N
- audit columns:
  created_date
  created_by
  last_modify_date
  last_modify_by

4. Use realistic Indonesian dummy data.

5. Passwords must use bcrypt hash (NOT plain text).

6. File naming convention:
